normally when terminal is run it is in a 'cooked mode'.
	raw mode

	cooked mode

read function loop
	STDIN_FILENO variable  and read() funtion are from unistd.h, 

	STDIN_FILENO is a defined file descriptor, as in it holds stdin(standard input) while the program is running

	read(<file descripter being read from>,<buffer storing what is read>,<number of bytes being read>) read returns the number of bytes read in file descriptor parameter

	so what this loop does is constantly get the first byte(the 1 param) of the terminal input(the STDIN_FILENO param) and store it in the c memory buffer(the &c memory address param)
